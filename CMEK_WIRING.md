# CMEK Wiring — Post-Deployment Service Agent IAM Bindings

Generated by Architex Landing Zone Wizard.
Run these commands AFTER `terraform apply` creates the workload projects,
since service agents require project numbers.

## Prerequisites

1. All workload projects must exist (run project factory stage first)
2. You need `roles/cloudkms.admin` on the security project
3. Set these variables:

```bash
export KMS_PROJECT="acme-lza1-sec-core-0"
export KEYRING_US_CENTRAL1="kr-us-central1"
export LOCATION_US_CENTRAL1="us-central1"
export KEYRING_US_EAST1="kr-us-east1"
export LOCATION_US_EAST1="us-east1"
```

## Project: app-dev-0

```bash
# Get project number:
export PROJECT_NUMBER_app_dev_0=$(gcloud projects describe acme-lza1-app-dev-0 --format="value(projectNumber)")

# Cloud Run container images and revisions
gcloud kms keys add-iam-policy-binding cloudrun \
  --keyring=kr-us-central1 --location=us-central1 --project=$KMS_PROJECT \
  --member="serviceAccount:service-$PROJECT_NUMBER_app_dev_0@serverless-robot-prod.iam.gserviceaccount.com" \
  --role="roles/cloudkms.cryptoKeyEncrypterDecrypter"

```

## Verification

After running the above commands, verify IAM bindings:

```bash
gcloud kms keys get-iam-policy <KEY_NAME> \
  --keyring=kr-us-central1 --location=us-central1 --project=$KMS_PROJECT
```

Check that each service agent has `roles/cloudkms.cryptoKeyEncrypterDecrypter`.
